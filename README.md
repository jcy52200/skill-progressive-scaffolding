# Progressive Scaffolding Skill

## 📌 简介
**渐进式脚手架技能** 是一个为 AI 助手设计的交互框架，旨在通过**结构化选项**和**动态提问**，帮助用户将模糊、复杂的需求逐步转化为清晰、可落地的方案。它像一位资深咨询顾问，不问废话，只问关键，与用户共同“搭建”出最终成果。

该技能特别适合产品经理、开发者、需求分析师、项目经理等需要频繁处理模糊需求的角色。

---

## ✨ 核心特性

| 特性 | 描述 |
|------|------|
| **🧩 结构化交互** | 全程使用 A/B/C 选项提问，用户只需做选择，而非填空题，降低认知负担。 |
| **🧠 认知脚手架** | 每次只问一个最关键问题，根据用户选择动态调整后续路径，精准采集需求。 |
| **🔄 需求共创** | 信息收齐后提供预览方案，与用户“认知对齐”，反复修改直至满意。 |
| **💾 知识资产沉淀** | 自动生成唯一命名的 Markdown 文件（时间戳+随机数），永不覆盖，所有方案可追溯、可复用。 |
| **🌐 联网知识增强** | 支持在提问或生成方案时检索最新网络资料（需平台支持），确保方案时效性与权威性。 |
| **📁 模板系统** | 预置多领域模板（软件开发、活动策划等），输出格式统一专业，亦可自定义。 |
| **🗂️ 可扩展架构** | 支持通过 `knowledge/` 知识库、`scripts/` 脚本、`config.yaml` 配置等深度定制。 |

## 📦 安装方法

### 1. 放置技能文件夹
将本技能文件夹（命名为 `progressive-scaffolding`）放入 AI 助手的技能扫描路径中：

- **Claude Desktop**  
  `~/.claude/skills/` （macOS/Linux）  
  `%APPDATA%\Claude\skills\` （Windows）

- **项目级路径**（与团队共享）  
  在你的项目根目录下创建 `.claude/skills/` 并将文件夹放入其中。

### 2. 确认结构
确保文件夹结构如下：
```
progressive-scaffolding/
├── SKILL.md                # 核心指令（AI 读取）
├── templates/              # 模板文件夹
├── knowledge/              # 知识库文件夹（可选）
└── scripts/                # 脚本文件夹（可选）

```

### 3. 启用技能
重启 AI 客户端，技能即被自动加载。当用户提出复杂任务（如“我要做一个点单系统”）时，AI 将自动激活该技能并开始轮询。

## 🚀 使用方法

### 与 AI 的典型交互流程

1. **用户提出需求**（例如：“我想开发一个奶茶店点单小程序。”）
2. **AI 识别任务**，自动匹配 `templates/` 中对应模板（若无则使用默认模板），并开始轮询。
3. **AI 依次提问**，每次给出 A/B/C 选项，用户选择并可选添加备注。
4. **过程中可要求检索**（如输入“检索一下微信小程序的开发成本”），AI 联网获取参考信息。
5. **信息收集完毕**，AI 生成预览方案，用户确认或提出修改。
6. **最终方案** 以唯一命名的 Markdown 文件保存于当前目录，AI 告知文件路径。

### 示例对话

> **用户**：我要做一个基于 Spring Boot 的饮品点单系统。  
> **AI**（检测到中文，匹配 `software-dev` 模板）：好的，我们开始规划。首先，系统的主要使用对象是？  
> A. 顾客（点单前端）  
> B. 店员（后台管理）  
> C. 两者都需要  
> D. 仅提供 API 接口  
> （您可在选项后添加备注）  
> **用户**：C，主要是给顾客用，但后台需要基础管理功能。  
> **AI**：明白。关于数据库选型，需要我为您检索一下 MySQL 和 PostgreSQL 的最新对比吗？（输入“检索”即可）  
> **用户**：检索一下。  
> **AI**（联网检索后）：根据 2026 年 3 月的 DB-Engines 排名，MySQL 在易用性上占优，PostgreSQL 在复杂查询上更强。您更侧重？  
> A. 快速开发（推荐 MySQL）  
> B. 复杂数据分析（推荐 PostgreSQL）  
> ...  
> （继续轮询...）  
> **AI**（信息收齐后生成预览）：这是初步方案概要，请确认...  
> **用户**：确认无误。  
> **AI**：已将完整方案保存为 `方案_20260321_153022_8f3a.md`，位于当前文件夹。

## 📂 文件结构详解

| 文件/文件夹 | 用途 |
|------------|------|
| `SKILL.md` | **核心指令**，AI 读取此文件理解技能逻辑。**请勿删除或重命名**。 |
| `templates/` | **模板系统**，按领域存放 `questions.yaml`（可选）和 `output-template.md`（输出格式）。 |
| `knowledge/` | **知识库**，存放行业报告、技术文档等，供 AI 按需引用（需 RAG 支持）。 |
| `scripts/` | **可执行脚本**，用于数据验证、外部 API 调用、格式转换等扩展功能。 |

---

## 🔧 扩展指南

### 添加自定义模板
1. 在 `templates/` 下新建一个文件夹，如 `my-domain/`。
2. 创建 `output-template.md`，定义输出格式（可使用 `{变量名}` 占位）。
3. 可选创建 `questions.yaml`，预置该领域常见问题（格式参考下文）。
4. AI 将根据用户描述的关键词自动匹配，或用户可手动输入“使用 my-domain 模板”。

**`questions.yaml` 示例**：
```yaml
first_question: "项目主要目标？"
options:
  A: "提升用户体验"
  B: "降低开发成本"
  C: "快速上线"
```

### 添加知识库
将 PDF、Markdown 等文档放入 `knowledge/` 对应子目录（如 `knowledge/software-dev/`）。若平台支持 RAG，AI 可在需要时检索这些文档以增强回答。

### 编写脚本
在 `scripts/` 下按功能划分子目录，例如：
- `scripts/validation/validate_date.py`：验证用户输入日期格式。
- `scripts/generation/generate_pdf.py`：将 Markdown 转换为 PDF。
AI 可在 SKILL.md 中通过指令调用这些脚本（需平台支持执行权限）。

## 📝 示例输出文件

生成的 Markdown 文件示例（基于 `software-dev` 模板）：

```markdown
# 奶茶店点单系统技术方案

## 1. 需求背景
开发一款基于 Spring Boot 的饮品点单系统，主要面向顾客提供点单功能，同时为店员提供基础后台管理。

## 2. 技术选型

- **数据库**：MySQL 8.0
- **前端技术**：微信小程序
- **部署方式**：阿里云 ECS

## 3. 核心模块
- 用户登录/注册
- 商品浏览与搜索
- 购物车与下单
- 订单管理
- 后台商品管理
- 基础数据统计

## 4. 数据模型
- 用户表、商品表、订单表、订单详情表、购物车表

## 5. 开发计划
- 第1周：环境搭建、数据库设计
- 第2-3周：后端接口开发
- 第4周：小程序前端联调
- 第5周：测试与部署
```
